# Voice Transcriber

A local web app that transcribes audio and video files with automatic speaker detection. Upload a recording or record live, get a clean transcript with speaker labels and timestamps, and copy it straight into ChatGPT.

Runs entirely on your machine. No cloud hosting, no accounts beyond the API keys.

## Features

**Transcription**
- Upload audio/video files (MP4, MOV, MP3, WAV, M4A, and more)
- Automatic speaker diarization — identifies who said what
- Click any speaker name to rename them (e.g. "Speaker 1" → "John")
- Timestamps on every utterance

**Voice Recognition**
- Automatic speaker identification using voice fingerprints
- When you rename a speaker, their voice embedding is saved using [resemblyzer](https://github.com/resemble-ai/Resemblyzer)
- On future transcriptions, speakers are automatically matched to known voices using cosine similarity
- Self-improving — correcting a wrong match updates the voiceprint
- No external API calls — all voice matching runs locally

**Live Recording**
- Record audio directly in the browser with one tap
- Progressive backup downloads every 30 seconds — if the browser crashes or your phone dies, you still have the recording saved locally
- Backups are named with a shared timestamp prefix so they group together in your Downloads folder
- On stop: saves the final audio file locally AND sends it to the server for transcription
- Auto-downloads a transcript `.txt` file after transcription completes
- Uses `.m4a` format on iPhone Safari, `.webm` on Chrome

**Audio Playback**
- Built-in audio player synced to the transcript
- Click any utterance to jump to that point and start playing
- Current utterance is highlighted and auto-scrolled during playback

**Search**
- Global search across all transcripts from the home page
- In-transcript search with match counter and up/down navigation

**AI Summaries & Action Items**
- Each transcript gets a 2-3 sentence summary generated by GPT-5
- Action items are automatically extracted from the conversation
- Action items are interactive: accept (checkmark), dismiss (strikethrough), or delete
- Click an action item to highlight the referenced section in the transcript
- Summaries and action items are stored in the transcript JSON — the API is only called once per transcript

**Export & Copy**
- "Copy for ChatGPT" — copies a formatted transcript with speaker names, timestamps, duration, and date
- Export as `.txt` or `.srt` (subtitle format)

**Usage & Cost Tracking**
- Dashboard showing this month and all-time usage
- Breakdown of Deepgram and GPT AI costs separately
- Per-file cost breakdown
- Deepgram credit remaining with progress bar and estimated months remaining

**Mobile Friendly**
- Dark mode UI optimized for iPhone Safari
- Touch-friendly controls, responsive layout
- Access from your phone over local network

## Prerequisites

- Python 3.11+
- ffmpeg
- A Deepgram account (free $200 credit on signup)
- An OpenAI API key (for GPT-5 summaries + action items)

## Setup

```bash
# Clone the repo
git clone https://github.com/Kina0110/voice-transcriber.git
cd voice-transcriber

# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate        # Mac/Linux
# .venv\Scripts\activate         # Windows

# Install dependencies
pip install -r requirements.txt

# Install ffmpeg (if you don't have it)
brew install ffmpeg              # Mac
# sudo apt install ffmpeg        # Ubuntu/Debian
# winget install ffmpeg           # Windows

# Set up API keys
cp .env.example .env
# Edit .env and add your keys
```

### `.env` file

```
DEEPGRAM_API_KEY=your_deepgram_key_here
OPENAI_API_KEY=your_openai_key_here
```

- **DEEPGRAM_API_KEY** (required) — for transcription and speaker diarization
- **OPENAI_API_KEY** (optional) — for AI summaries and action item extraction. Leave blank to skip.

## Running

```bash
source .venv/bin/activate
uvicorn app:app --host 0.0.0.0 --port 8000
```

Open **http://localhost:8000** in your browser.

### Accessing from your phone

The app binds to `0.0.0.0:8000`, so any device on your local network can reach it:

1. Find your computer's local IP: `ifconfig | grep "inet " | grep -v 127.0.0.1`
2. On your phone, open Safari and go to `http://<your-ip>:8000`

Alternatively, install [Tailscale](https://tailscale.com/download) on both devices for a more reliable connection.

## Live Recording Backup Structure

When you use live recording, files are saved to your browser's Downloads folder with a shared timestamp prefix:

```
rec-2026-02-21T14-30-05-backup-1.m4a       <- auto-saved at 0:30
rec-2026-02-21T14-30-05-backup-2.m4a       <- auto-saved at 1:00
rec-2026-02-21T14-30-05-final.m4a          <- saved when you hit Stop
rec-2026-02-21T14-30-05-transcript.txt     <- auto-saved after transcription
```

Each backup contains the **entire recording up to that point** (not just the last 30 seconds), so any single backup file is self-contained and playable.

## Tech Stack

- **Backend:** Python / FastAPI
- **Frontend:** Vanilla HTML / CSS / JS (no build step, no framework)
- **Transcription:** Deepgram Nova-3 with speaker diarization
- **Summaries:** OpenAI GPT-5
- **Voice Recognition:** resemblyzer (local, no API)
- **Audio processing:** ffmpeg
- **Storage:** JSON files on disk (no database needed)
